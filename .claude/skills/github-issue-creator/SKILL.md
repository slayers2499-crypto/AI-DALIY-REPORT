# GitHub 이슈 생성 스킬

현재 작업 중인 GitHub 리포지토리에 새로운 이슈를 빠르게 생성할 수 있습니다.

## 사용 방법

이슈를 생성할 때 다음 정보를 제공하세요:

### 필수 정보
- **제목**: 이슈의 간단한 설명

### 선택 정보
- **설명**: 상세한 내용 (선택사항)
- **레이블**: 이슈를 분류하는 태그 (예: bug, feature, documentation)
  - 리포지토리에 존재하는 레이블만 사용 가능
  - 여러 개는 쉼표로 구분: `bug, urgent`
- **담당자**: 이슈를 담당할 사람의 GitHub 사용자명
  - `@me` 또는 `@username` 형식 사용

## 사용 예시

### 예시 1: 버그 리포트
```
제목: 로그인 페이지 버튼 레이아웃 수정
설명: 모바일 화면에서 로그인 버튼이 제대로 보이지 않습니다.
레이블: bug, mobile
담당자: @username
```

### 예시 2: 기능 요청
```
제목: 다크 모드 지원
설명: 사용자 경험을 개선하기 위해 다크 모드 옵션을 추가해주세요.
레이블: enhancement
담당자: @me
```

### 예시 3: 문서화 작업
```
제목: 설치 가이드 업데이트
레이블: documentation
```

## 결과

- 이슈가 생성되면 GitHub 이슈 URL을 반환합니다
- 지정한 레이블과 담당자가 이슈에 자동으로 적용됩니다

## 주의사항

- `gh` CLI가 설치되고 GitHub 계정으로 인증되어 있어야 합니다
- 현재 디렉토리가 Git 리포지토리의 클론이어야 합니다
- 사용하려는 레이블은 리포지토리에 존재해야 합니다 (없으면 오류 발생)
- 담당자 이름도 리포지토리에 접근 권한이 있는 사용자여야 합니다

## 레이블 확인

사용 가능한 레이블을 확인하려면:
```
gh label list
```
