# 🔍 프로젝트 점검 스킬

자동화된 프로젝트 감시 및 개선사항 발견 시스템입니다.

## 목표

현재 Git 저장소의 코드베이스를 다각도로 분석하여 다음을 식별합니다:
- 🐛 버그 및 보안 문제
- 🚀 성능 최적화 기회
- 📚 문서화 부족
- 🔒 보안 취약점
- 📊 코드 품질 문제
- ⚙️ 설정 누락

## 점검 항목

### 1️⃣ 프로젝트 구조 분석
```
- 폴더/파일명 확인 (오타, 일관성)
- 필수 파일 존재 여부 (README.md, LICENSE, .gitignore 등)
- 디렉토리 깊이 및 복잡도
- 임시 파일 및 로그 파일 존재 여부
```

### 2️⃣ Git 설정 검증
```
- 원격 저장소 설정 확인
- 토큰/자격증명 노출 여부
- 커밋 메시지 형식 일관성
- 브랜칭 전략 확인
```

### 3️⃣ 문서화 점검
```
- README.md 존재 및 내용 확인
- CLAUDE.md 존재 여부
- API 문서 존재 여부
- 설정 가이드 확인
```

### 4️⃣ 보안 분석
```
- 노출된 비밀키/토큰 (API keys, tokens, credentials)
- 민감한 데이터 추적 여부
- .env 파일 .gitignore 포함 여부
- 의존성 취약점 (if applicable)
```

### 5️⃣ 성능 평가
```
- 파일 크기 및 라인 수 분석
- 스크립트 실행 시간 (available)
- 메모리 사용량 (if trackable)
- 병렬화 가능 영역 식별
```

### 6️⃣ 코드 품질
```
- 중복 코드 감지
- 에러 처리 부족
- 하드코딩된 값 (매직 넘버, 경로)
- 테스트 커버리지
```

### 7️⃣ 설정 검증
```
- config.json 또는 설정 파일 존재
- 환경 변수 사용 여부
- 스케줄 설정 검증
- API 설정 완성도
```

### 8️⃣ 프로세스 검증
```
- CI/CD 설정 (workflow files)
- 테스트 자동화
- 배포 프로세스
- 모니터링/로깅 설정
```

## 실행 방법

### 기본 점검
```bash
프로젝트 점검해줘
```

### 특정 영역만 점검
```bash
보안 문제만 점검해줘
성능 최적화 기회를 찾아줘
문서화 부족한 부분을 찾아줘
```

### 심도 있는 점검
```bash
코드베이스 전체 감시해줘
```

## 출력 형식

### 발견 사항 요약
```
🔴 긴급 (Critical)
  - 보안 취약점
  - 데이터 손실 위험
  - 운영 중단 문제

🟡 높음 (High)
  - 성능 병목
  - 에러 처리 부족
  - 문서화 누락

🟠 중간 (Medium)
  - 코드 개선
  - 최적화 기회
  - 일관성 문제

🟢 낮음 (Low)
  - 스타일 개선
  - 작은 리팩토링
  - 문서 추가
```

### GitHub 이슈 자동 생성
발견된 각 항목에 대해:
- 제목: 명확한 문제 설명
- 설명: 문제점, 영향, 해결책
- 레이블: 심각도별 분류
- 우선순위: 즉시/단기/중기/장기

## 점검 주기

**권장 실행 주기:**
- 매주 1회: 일반적인 프로젝트
- 매월 1회: 안정적인 프로젝트
- 새 기능 추가/배포 전: 집중 점검

## 내부 동작

1. **정적 분석** (Static Analysis)
   - 파일 시스템 스캔
   - Git 설정 검증
   - 문서 존재 확인

2. **패턴 매칭** (Pattern Matching)
   - 하드코딩된 값 탐지
   - 비밀키 패턴 인식
   - 명명 규칙 확인

3. **휴리스틱 평가** (Heuristic Evaluation)
   - 파일 크기 분석
   - 복잡도 추정
   - 성능 병목 식별

4. **문제 분류** (Classification)
   - 심각도 판단
   - 범주 할당
   - **이슈 자동 생성** ✨

5. **자동 GitHub 이슈 등록**
   - 발견된 각 항목마다 GitHub 이슈 생성
   - 제목: 명확한 문제 설명
   - 설명: 문제점, 영향, 해결책 포함
   - 레이블: 심각도별 자동 분류
   - 중복 검사: 기존 이슈와 비교하여 중복 방지

## 주의사항

⚠️ **이 스킬은:**
- 정적 분석만 수행합니다 (실행 기반 테스트 아님)
- 프로젝트의 모든 문제를 찾지 못할 수 있습니다
- 휴리스틱 기반이므로 오탐지 가능성 있음
- 수동 코드 리뷰를 대체하지 않습니다

✅ **추가 검토 권장:**
- 보안 심화 감시
- 성능 프로파일링
- 사용자 피드백
- 실제 운영 모니터링

## 예상 결과

각 점검 실행 후:
- ✅ 발견된 개선사항 요약
- 📋 GitHub 이슈 자동 생성
- 📊 점검 리포트 생성
- 🎯 우선순위별 로드맵 제시
