# 📚 문서 생성 스킬

프로젝트에 필요한 문서를 현재 코드와 설정을 기반으로 자동 생성합니다.

## 언제 사용하나요?

- 프로젝트 문서가 없거나 오래됐을 때
- 새 팀원이 합류해 온보딩 문서가 필요할 때
- 배포 가이드, 보안 정책 등을 정리할 때

## 사용 방법

```
프로젝트 문서 생성해줘
README 만들어줘
배포 가이드 작성해줘
보안 정책 문서 만들어줘
전체 문서 생성해줘
```

## 생성 가능한 문서

### 📖 README.md
- 프로젝트 개요
- 시스템 요구사항
- 설치 및 실행 방법
- 주요 기능 목록
- 스크린샷 또는 예시
- 기여 가이드

### 🚀 DEPLOY.md
- 환경 구성
- 스케줄 작업 설정
- 의존성 설치
- 첫 실행 방법
- 문제 해결 (Troubleshooting)

### 🔒 SECURITY.md
- 보안 정책
- 취약점 보고 방법
- API 키 관리 방법
- 금지 사항

### ⚙️ CONFIG.md
- 설정 파일 설명
- 환경 변수 목록
- 커스터마이징 방법
- 기본값 및 옵션

### 📋 CHANGELOG.md
- 버전별 변경 이력
- 새 기능, 버그 수정, 보안 패치

## 실행 절차

1. **현재 프로젝트 구조 파악**
   - 파일/폴더 스캔
   - 기존 문서 확인
   - SKILL.md 파일 분석

2. **내용 분석**
   - git log에서 변경 이력 추출
   - 설정 파일에서 옵션 목록 추출
   - 코드에서 기능 목록 추출

3. **문서 생성**
   - 한국어 우선, 기술용어는 영어 병기
   - Markdown 형식, GitHub 렌더링 최적화
   - 실제 경로/명령어 포함

4. **검증**
   - 링크 유효성 확인
   - 명령어 실행 가능성 확인

## 주의사항

- 민감한 정보(토큰, 비밀번호)는 문서에 포함하지 않음
- 환경별 설정은 `.env.example`로 분리
- 자동 생성 후 사람이 검토 권장
