# 📄 문서 최적화 스킬

3문서 역할을 분리하고, 중복을 제거하며, 낡은 문서를 정리합니다.

## 3문서 역할 원칙

| 문서 | 독자 | 담아야 할 것 |
|------|------|-------------|
| **SOUL.md** | Claude | WHO(사용자), WHY(목적), 판단 원칙, 품질 기준 |
| **CLAUDE.md** | Claude | HOW(경로, 운영 규칙, 스킬 트리거, 환경 제약) |
| **README.md** | 사람 | WHAT(개요), 시작 방법, 명령 목록, 설정 방법 |

## 실행 흐름

### 1단계 — 현황 분석
- 현재 문서 목록 파악 (`SOUL.md` 존재 여부 포함)
- 각 문서의 내용과 역할 확인
- 중복 항목 식별

### 2단계 — 역할 분리
각 항목을 아래 기준으로 분류:

- **SOUL.md로 이동**: 사용자 설명, 목적, 품질 원칙, 이슈 판단 기준, 응답 원칙
- **CLAUDE.md로 이동**: 경로, 환경 제약, 금지 사항, 스킬 트리거
- **README.md로 이동**: 설치 방법, 시작 명령, 리포트 구성, 설정 방법
- **삭제 대상**: 위 세 문서에 이미 있는 내용과 100% 중복인 별도 파일

### 3단계 — 최적화 3원칙 적용

1. **SSOT** (Single Source of Truth) — 동일 내용이 두 곳에 있으면 하나만 남긴다
2. **Minimum** — 필요한 내용만. 설명보다 사실, 문단보다 표
3. **Freshness** — 낡은 날짜·경로·이슈 번호는 현재 상태로 갱신

### 4단계 — 문서 작성 및 정리
- SOUL.md 없으면 신규 생성
- CLAUDE.md·README.md 재작성 (중복 제거, 역할 외 내용 이동)
- 중복 파일 삭제 (git rm 포함)
- git commit

## 트리거

```
프로젝트 문서 최적화해줘
3문서 역할 분리해줘
문서 중복 정리해줘
```

## 결과물

- `SOUL.md` — WHO·WHY·판단 원칙
- `CLAUDE.md` — 운영 규칙, `@SOUL.md` 참조 포함
- `README.md` — 인간 독자용 온보딩
- 중복 파일 삭제 + git commit
