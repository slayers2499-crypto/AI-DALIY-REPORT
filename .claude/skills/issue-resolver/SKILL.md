# 🔧 이슈 해결 스킬

GitHub에 등록된 이슈를 분석하고, 실제 파일을 수정/생성하여 이슈를 자동으로 해결합니다.

## 언제 사용하나요?

- 등록된 이슈들을 실제로 해결하고 싶을 때
- 특정 이슈를 골라 해결하고 싶을 때
- 문서/설정 관련 이슈를 빠르게 처리할 때
- 우선순위 순으로 이슈를 해결할 때

## 사용 방법

### 전체 해결 (우선순위 순)
```
이슈들 해결해줘
열린 이슈 처리해줘
이슈 자동 해결해줘
```

### 특정 이슈만
```
#27 이슈 해결해줘
긴급 이슈만 해결해줘
문서 관련 이슈 처리해줘
```

### 범위 지정
```
#18, #19, #20, #21 해결해줘
documentation 이슈들 처리해줘
bug 이슈만 해결해줘
```

## 해결 가능한 이슈 유형

### ✅ 즉시 자동 해결 가능
| 유형 | 예시 이슈 | 해결 방법 |
|------|----------|----------|
| 문서 파일 추가 | README.md, DEPLOY.md, SECURITY.md | 파일 생성 |
| 설정 파일 추가 | config.json, .gitignore, CLAUDE.md | 파일 생성 |
| LICENSE 추가 | LICENSE 파일 | MIT 라이선스 생성 |
| CHANGELOG 추가 | CHANGELOG.md | 변경 이력 생성 |

### 🔶 분석 후 해결 가능
| 유형 | 예시 이슈 | 해결 방법 |
|------|----------|----------|
| HTML 개선 | ARIA 태그, 반응형 CSS | 파일 수정 |
| 보안 문제 | 토큰 노출, .gitignore | 파일 수정 + 재설정 |
| 성능 최적화 | async/defer, 병렬화 | 코드 수정 |

### ❌ 수동 처리 필요 (스킬이 안내만 제공)
| 유형 | 이유 |
|------|------|
| 이메일 배송 연동 | 외부 서비스 계정 필요 |
| CI/CD 설정 | GitHub Actions 권한 필요 |
| 모니터링 설정 | 외부 서비스 연동 필요 |

## 실행 절차

### 1단계: 이슈 목록 조회 및 분류
```powershell
gh issue list --state open --limit 50
```
이슈를 다음 기준으로 분류:
- **즉시 해결**: 파일 생성/수정으로 완료
- **분석 필요**: 기존 코드 분석 후 수정
- **수동 필요**: 외부 연동이나 권한 필요

### 2단계: 우선순위별 해결
순서: 긴급 → 높음 → 중간 → 낮음

각 이슈 해결 시:
1. 이슈 내용 상세 확인 (`gh issue view <번호>`)
2. 관련 파일 분석
3. 실제 수정/생성 실행
4. 변경사항 git add + commit
5. 이슈 클로즈 + 해결 코멘트 작성

### 3단계: 이슈 클로즈
```powershell
gh issue close <번호> --comment @'
## ✅ 해결 완료

**해결 방법:** [수행한 작업 요약]
**변경 파일:** [수정/생성된 파일 목록]
**커밋:** [커밋 해시]

자동 해결 by issue-resolver 스킬
'@
```

### 4단계: 완료 요약
```
✅ 해결 완료: 8개
⏭️ 건너뜀 (수동 필요): 4개
❌ 실패: 0개

📊 해결 내역:
  #18 README.md 추가 ✅
  #19 .gitignore 추가 ✅
  #21 CLAUDE.md 추가 ✅
  ...
```

## 주의사항

- 이슈 해결 전 항상 현재 파일 상태 확인
- 기존 파일 수정 시 백업 또는 git diff 확인
- 보안 관련 이슈(#27)는 민감하므로 신중하게 처리
- 각 이슈 해결 후 즉시 commit (rollback 가능하도록)
- 수동 처리 필요 항목은 이슈에 가이드 코멘트 남기고 오픈 유지
