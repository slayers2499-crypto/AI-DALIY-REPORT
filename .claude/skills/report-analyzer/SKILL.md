# 🔍 리포트 분석 스킬

생성된 HTML 리포트의 품질을 자동으로 검증하고 개선점을 제안합니다.

## 언제 사용하나요?

- 생성된 리포트가 제대로 만들어졌는지 확인할 때
- 리포트 품질을 수치로 파악하고 싶을 때
- C-레벨 공유 전 최종 검토할 때

## 사용 방법

```
리포트 분석해줘
오늘 리포트 품질 확인해줘
리포트 점검해줘
report_20260611.html 분석해줘
```

## 검증 항목

### 📊 콘텐츠 품질
- [ ] 수집 기사 수 (5개 이상)
- [ ] 섹션 완성도 (6개 모두 존재)
- [ ] 각 섹션 최소 분량 (200자 이상)
- [ ] Executive Summary 3줄 이상
- [ ] 전략적 시사점 액션 아이템 5개 이상

### 📅 최신성
- [ ] 기사 날짜가 7일 이내인지 확인
- [ ] 오래된 기사(3개월 이상) 비율 확인
- [ ] 오늘 날짜 언급 여부

### 🎨 HTML 품질
- [ ] 모바일 반응형 CSS 존재 (@media query)
- [ ] 제목, 섹션 구조 올바름
- [ ] 한국어 lang 태그 (`<html lang="ko">`)
- [ ] CSS 변수 사용 일관성
- [ ] 인코딩 `charset=UTF-8` 확인

### 🔗 출처 신뢰성
- [ ] 출처 URL 포함 여부
- [ ] 신뢰 가능한 도메인 비율 (TechCrunch, Reuters, Axios 등)
- [ ] 오피셜 채널 비율

## 점수 산정

| 항목 | 배점 | 기준 |
|------|------|------|
| 수집 기사 수 | 20점 | 7개 이상 = 만점 |
| 섹션 완성도 | 20점 | 6/6 = 만점 |
| 콘텐츠 분량 | 20점 | 섹션당 300자 이상 = 만점 |
| 최신성 | 20점 | 모든 기사 7일 이내 = 만점 |
| HTML 품질 | 20점 | 반응형 + 구조 완성 = 만점 |

**총점 80점 이상:** 공유 가능  
**총점 60~79점:** 개선 후 공유  
**총점 60점 미만:** 재생성 권장

## 출력 예시

```
📊 리포트 분석 결과: report_20260611.html

콘텐츠 품질:  18/20 ✅
섹션 완성도:  20/20 ✅
콘텐츠 분량:  16/20 ⚠️  (트렌드 섹션 분량 부족)
최신성:       20/20 ✅
HTML 품질:    18/20 ✅

🏆 총점: 92/100 — 공유 가능!

💡 개선 제안:
1. 트렌드 분석 섹션에 내용 추가 권장 (현재 180자 → 목표 300자)
2. 모바일 480px 중간 브레이크포인트 추가 고려
```

## 주의사항

- HTML 파일 크기 기준: 20KB~100KB 정상 범위
- 100KB 초과 시 이미지/중복 콘텐츠 확인 필요
- 생성 시간 기록 없을 경우 파일 수정 시간 사용
