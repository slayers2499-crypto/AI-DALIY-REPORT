### #1 수정: 스케줄 작업이 08:00이 아닌 08:02에 실행됨 — 크론 시간 오차
**우선순위:** 긴급  
**상태:** 열림  
**타입:** 버그  

**문제:**  
스케줄 작업 `daily-ai-trend-report`가 cronExpression `0 8 * * *`로 설정되었지만 08:02 AM에 실행됩니다.

**영향:**
- 리포트 배달이 의도한 시간보다 2분 늦음
- 다운스트림 프로세스 또는 예약된 의존성에 시간 문제 발생 가능

**해결책:**
- 2분 오차의 원인 조사 (시스템 시작 지연, 클록 동기화, 작업 큐)
- cronExpression 조정 또는 시스템 시간대 확인
- 실제 실행 시간 문서화

### #2 개선: C:\Temp에서 영구 디렉토리로 리포트 저장 위치 변경
**우선순위:** 긴급  
**상태:** 열림  
**타입:** 개선  

**문제:**  
리포트 파일이 임시 시스템 폴더인 `C:\Temp\AI-DALIY-REPORT\`에 저장되고 있습니다. Windows는 임시 디렉토리를 자동으로 정리합니다.

**영향:**
- 과거 리포트 손실 위험 (임시 폴더 정리)
- 프로덕션 환경에 부적합
- 사용자가 영구 저장소에 의존할 수 없음

**해결책:**
- 영구 디렉토리로 이동: `C:\Users\Admin\Documents\AI-Reports\` 또는 네트워크 드라이브
- 월별 하위 폴더 구조 생성: `reports/2026/06/report_20260611.html`
- 작업 프롬프트 SKILL.md에 새 경로 업데이트
- 디렉토리가 없으면 생성하는 로직 추가

### #3 기능: 일일 리포트에 이메일 배송 추가
**우선순위:** 높음  
**상태:** 열림  
**타입:** 개선  

**문제:**  
생성된 HTML 리포트가 로컬에 저장되지만 이해관계자에게 자동으로 전송되지 않습니다. C-레벨 임원들이 파일을 수동으로 다운로드해야 합니다.

**영향:**
- 사용자가 매일 새 리포트를 수동으로 확인해야 함
- 리포트가 의도한 대상자에게 자동으로 도달하지 않음
- 도입율 및 시간-인사이트 감소
- 푸시 알림 이점 부재

**해결책:**
- 이메일 배송 통합 (SMTP, SendGrid, 또는 클라우드 이메일 서비스)
- 수신자 이메일 주소 설정 추가 (config.json의 배열)
- 이메일 본문에 HTML 첨부 + 요약 텍스트 포함
- 리포트 생성 후 15분에 이메일 스케줄 (오전 8:15)
- 이메일 미리보기에 빠른 통계 포함

### #4 기능: 리포트 아카이브 및 검색 기능 구축
**우선순위:** 높음  
**상태:** 열림  
**타입:** 기능  

**문제:**  
과거 리포트에 접근하거나 과거 AI 트렌드를 검색할 방법이 없습니다. 각 리포트는 독립적입니다.

**영향:**
- 사용자가 여러 날짜 간 인사이트 비교 불가
- 리포트가 일회용이지, 누적 지식 기반이 아님
- 장기 트렌드 파악 어려움
- 패턴 인식을 위한 참고 자료 부재

**해결책:**
- 리포트 아카이브 인덱스 생성 (JSON: `archive_index.json`)
- HTML 대시보드 구축: `archive.html`에 모든 리포트 나열
- 날짜, 키워드, 회사, 주제별 검색
- 두 개의 리포트 나란히 비교
- 구성: `reports/2026/06/report_20260611.html` 등

**승인 기준:**
- 아카이브 대시보드 2초 내 로드
- 검색 결과 1초 내 표시
- 모바일 반응형

### #5 기능: 리포트 생성에 에러 처리 및 재시도 로직 추가
**우선순위:** 높음  
**상태:** 열림  
**타입:** 개선  

**문제:**  
현재 워크플로우에는 다음에 대한 에러 처리가 없습니다:
- 웹 검색 실패 / 레이트 제한
- WebFetch 타임아웃 또는 잘못된 URL
- 불완전한 데이터 수집
- 네트워크 오류
- HTML 생성 실패

단계 중 하나라도 실패하면 전체 리포트가 조용히 실패합니다.

**영향:**
- 사용자가 인식하지 못한 채 리포트 누락 또는 손상
- 부분 폴백 없음 (사용자는 아무것도 받지 못함)
- 실패 디버깅 불가
- 예약 작업이 조용히 실패할 수 있음

**해결책:**
- 작업 프롬프트에 try-catch 에러 처리 추가
- 실패한 API 호출에 재시도 로직 구현 (지수 백오프 포함 3회)
- 데이터가 불완전하면 에러 플래그가 있는 부분 리포트 생성
- 실패 추적을 위한 상세 로깅 추가
- 생성 실패 시 이메일 알림 전송
- "마지막 업데이트" 타임스탬프 포함하여 오래된 리포트 표시

### #6 기능: 다국어 리포트 생성 지원 추가
**우선순위:** 중간  
**상태:** 열림  
**타입:** 기능  

**문제:**  
리포트가 현재 한국어로만 생성됩니다. 국제 팀이 사용할 수 없습니다.

**영향:**
- 한국어 사용 팀 범위 벗어나 미치는 범위 제한
- 수동 번역 시 중복 노력
- 글로벌 조직의 가치 감소
- 비한국어 사용자 제외

**해결책:**
- 작업 프롬프트에 언어 설정 매개변수 추가
- 지원 언어: 한국어 (기본), 영어 (우선순위), 중국어, 일본어
- 작업 설정에 언어 선택기 추가
- HTML에 언어 태그 포함: `<html lang="ko">` 또는 `lang="en"`
- 별도 리포트 버전 또는 단일 이중 언어 버전 유지

### #7 QA: 모바일 기기에서 HTML 리포트 반응형 테스트
**우선순위:** 중간  
**상태:** 열림  
**타입:** QA/테스트  

**문제:**  
HTML 리포트가 flexbox/grid를 사용하지만 실제 모바일 기기에서 테스트되지 않았습니다.

**영향:**
- 리포트가 휴대폰/태블릿에 제대로 렌더링되지 않을 수 있음
- 모바일에서 본 경우 나쁜 사용자 경험
- CSS가 특정 화면 크기에서 깨질 수 있음

**테스트 계획:**
- iPhone 12 (375px 너비)
- iPad (768px 너비)
- Galaxy S21 (Android, 360px)
- Chrome 모바일, Safari 모바일
- 테스트 섹션: 헤더, 카드, 표, 푸터

**수정 이슈:**
- 뉴스 헤드라인 텍스트 오버플로우
- 섹션 헤더 보이지 않음
- 이미지 크기 조정 안 됨
- 버튼/링크 탭 타겟이 너무 작음

### #8 버그: 스케줄 작업의 시간대 설정 확인
**우선순위:** 중간  
**상태:** 열림  
**타입:** 버그/명확화  

**문제:**  
크론 표현식이 로컬 시간대에서 실행되지만 정확한 시간대가 불명확합니다. KST (UTC+9)인지 확인이 필요합니다.

**영향:**
- 리포트가 다른 지역에서 예상 외 시간에 실행될 수 있음
- 설명서에 시간대가 지정되지 않음
- 시스템이 다른 시간대로 이동할 경우 이식성 문제

**해결책:**
- SKILL.md에 시간대 문서화: "한국 표준 시간 (KST, UTC+9)에서 실행"
- 실제 실행 시간 확인 (08:00 AM KST, UTC 아님)
- 리포트를 실행하고 타임스탬프를 확인하여 테스트
- 이식성이 필요한 경우 시간대 매개변수화 고려

**영향:**

### #10 개선: 리포트 커스터마이징을 위한 설정 파일 추가
**우선순위:** 낮음  
**상태:** 열림  
**타입:** 개선  

**문제:**  
모든 설정이 작업 프롬프트에 하드코딩되어 있습니다. 커스터마이징하려면 사용자가 SKILL.md를 직접 편집해야 합니다.

**영향:**
- 기술이 아닌 사용자는 커스터마이징할 수 없음
- 프롬프트 편집으로 작업을 깨기 쉬움
- 코드형 설정 모범 사례 없음
- 설정이 여러 곳에 분산됨

**해결책:**
`config.json` 생성:
```json
{
  "search_keywords": ["AI", "machine learning", "LLM", "generative AI"],
  "language": "ko",
  "report_recipients": ["ceo@company.com", "strategy@company.com"],
  "num_articles": 7,
  "storage_path": "C:\\Users\\Admin\\Documents\\AI-Reports",
  "report_style": {
    "primary_color": "#2563EB",
    "theme": "light"
  },
  "timezone": "Asia/Seoul"
}
```

작업 프롬프트를 하드코딩된 값 대신 설정 파일을 읽도록 업데이트합니다.

### #11 기능: 리포트 지표 및 분석 대시보드 추가
**우선순위:** 낮음  
**상태:** 열림  
**타입:** 기능  

**문제:**  
리포트 효과성에 대한 가시성이 없습니다: 성공률, 사용자 참여, 콘텐츠 공감도.

**영향:**
- ROI 또는 효과성 측정 불가
- 어떤 콘텐츠/회사가 공감을 얻는지 알 수 없음
- 시간 경과에 따라 품질 개선할 데이터 없음
- 피드백 루프 부재

**해결책:**
- HTML에 분석 추적 추가 (페이지 뷰, 섹션 클릭)
- 다음을 보여주는 분석 대시보드 생성:
  - 일일 생성 성공률 %
  - 가장 자주 언급되는 회사/주제
  - 시간 경과에 따른 리포트 뷰 수
  - 인기 섹션 / 읽히지 않는 섹션
- 로그 지표: 생성 시간, 파일 크기, 오류
- 주간 요약 이메일 생성

### #12 기능: API 호출의 레이트 제한 처리 추가
**우선순위:** 낮음  
**상태:** 열림  
**타입:** 개선  

**문제:**  
WebSearch는 피크 시간 또는 연속 여러 호출 후 레이트 제한될 수 있습니다.

**영향:**
- 레이트 제한 시 리포트 생성이 조용히 실패
- 지수 백오프 또는 재시도 전략 없음
- 요청에 대한 큐 관리 없음

**해결책:**
- 레이트 제한 오류 감지 (HTTP 429)
- 지수 백오프 재시도 전략 구현
- 요청 큐잉/스로틀링 추가
- 레이트 제한 히트 로깅하여 패턴 추적
- 이전 결과를 폴백으로 캐싱 고려
